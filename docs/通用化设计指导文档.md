## 一、设计目标

### 1.1 核心思想

**插件只管自己的，其他全靠读取**

**文件夹隔离原则**：
- ✅ 插件生成的所有内容都放在 `工作台/` 文件夹内
- ✅ 日计划：`工作台/日计划/YYYY-MM-DD-日计划.md`
- ✅ 周计划：`工作台/周计划/YYYY-WXX-本周计划.md`
- ✅ 生成的统计文档：`工作台/逾期事项.md`、`工作台/已完成事项.md` 等
- ❌ 用户的文件夹：SAP/、日记/、节点/、待整理/ 等（不应硬编码，不应混入）

**数据读取原则**：
- ❌ 用户的标签：#知识、#项目、#节点 等（不应硬编码）
- ✅ 通过配置或动态发现读取用户数据

### 1.2 通用性要求

| 要求 | 说明 |
|------|------|
| 领域无关 | 不限于 SAP 学习，任何领域都能用 |
| 结构无关 | 不限于特定的文件夹结构 |
| 标签无关 | 不限于特定的标签体系 |
| 动态发现 | 启动时自动读取用户的文件夹和标签 |

### 1.3 文件夹结构

```
用户知识库/
├── 工作台/                    ← 插件生成的所有内容都在这里
│   ├── 日计划/               ← 日计划
│   │   ├── 2026-05-28-日计划.md
│   │   └── ...
│   ├── 周计划/               ← 周计划
│   │   ├── 2026-W22-本周计划.md
│   │   └── ...
│   ├── 逾期事项.md           ← 生成的统计文档
│   ├── 已完成事项.md
│   ├── 进行中事项.md
│   └── 文档概览.md
├── 事项/                     ← 用户的事项（通过配置读取）
├── 日记/                     ← 用户的日记（不涉及）
├── SAP/                      ← 用户的其他文件夹（不涉及）
└── ...
```

**配置原则**：
- P0 阶段只做简单的键值对配置
- 不做抽象的模块配置系统（标签页/状态栏的自定义放到 P2）

- **工作台根目录单一配置**：所有插件生成的内容都在 `工作台/` 下

## 四、动态发现机制

### 4.1 发现与配置的关系

**原则：配置优先，发现作为默认值来源**

- 用户配置的值优先使用
- 动态发现的结果作为配置页面的候选值展示
- 如果用户没有配置，使用动态发现的结果作为默认值

### 4.2 文件夹发现（带缓存）

```typescript
class FolderDiscovery {
  private cache: string[] = [];
  private lastScan: number = 0;
  private readonly CACHE_TTL = 60000; // 1分钟缓存

  discover(app: App): string[] {
    const now = Date.now();
    if (this.cache.length > 0 && now - this.lastScan < this.CACHE_TTL) {
      return this.cache;
    }

    const folders = new Set<string>();
    app.vault.getMarkdownFiles().forEach(file => {
      const parts = file.path.split('/');
      if (parts.length > 1) {
        folders.add(parts[0]);
      }
    });

    this.cache = Array.from(folders);
    this.lastScan = now;
    return this.cache;
  }

  invalidate(): void {
    this.cache = [];
  }
}
```

### 4.3 标签发现（使用 metadataCache）

```typescript
class TagDiscovery {
  private cache: string[] = [];

  discover(app: App): string[] {
    const tags = new Set<string>();
    // 使用 metadataCache 而非遍历文件
    const files = app.vault.getMarkdownFiles();
    files.forEach(file => {
      const cache = app.metadataCache.getFileCache(file);
      if (cache?.frontmatter?.tags) {
        cache.frontmatter.tags.forEach((tag: string) => tags.add(tag));
      }
    });
    this.cache = Array.from(tags);
    return this.cache;
  }
}
```

### 4.4 Frontmatter 字段发现（带白名单过滤）

```typescript
// 白名单：排除 Obsidian 内置字段
const IGNORED_FIELDS = new Set([
  'cssclass', 'publish', 'aliases', 'tags',
  'created', 'updated', 'date'
]);

class FieldDiscovery {
  discover(app: App): string[] {
    const fields = new Set<string>();
    app.vault.getMarkdownFiles().forEach(file => {
      const cache = app.metadataCache.getFileCache(file);
      if (cache?.frontmatter) {
        Object.keys(cache.frontmatter)
          .filter(key => !IGNORED_FIELDS.has(key))
          .forEach(key => fields.add(key));
      }
    });
    return Array.from(fields);
  }
}
```

## 六、模板系统通用化

### 6.1 模板存储策略

**采用简单策略**：模板存储为插件目录下的 `.md` 文件，用户可以直接编辑

```
.obsidian/plugins/worktop/templates/
├── daily-plan.md      # 日计划模板
├── weekly-plan.md     # 周计划模板
└── topic.md           # 事项模板
```

### 6.2 模板变量

模板中使用 `{{变量名}}` 作为占位符：

```markdown
# {{date}} 日计划

> 本周计划：[[{{weeklyPlanPath}}]]

## 今日要事

- [ ] 任务 1
```

**可用变量**：
- `{{date}}` - 当前日期
- `{{weekNum}}` - 周数
- `{{year}}` - 年份
- `{{weeklyPlanPath}}` - 周计划路径
- `{{topicType}}` - 事项类型
- `{{title}}` - 标题

### 6.3 模板初始化

首次启动时，如果模板文件不存在，自动从默认模板创建：

```typescript
async function initTemplates(pluginPath: string): Promise<void> {
  const templateDir = `${pluginPath}/templates`;
  if (!await this.app.vault.adapter.exists(templateDir)) {
    await this.app.vault.createFolder(templateDir);
  }

  // 创建默认模板
  const templates = {
    'daily-plan.md': DEFAULT_DAILY_PLAN_TEMPLATE,
    'weekly-plan.md': DEFAULT_WEEKLY_PLAN_TEMPLATE,
    'topic.md': DEFAULT_TOPIC_TEMPLATE,
  };

  for (const [filename, content] of Object.entries(templates)) {
    const path = `${templateDir}/${filename}`;
    if (!await this.app.vault.adapter.exists(path)) {
      await this.app.vault.create(path, content);
    }
  }
}
```

### 10.2 领域统计逻辑

**统计方式**：通过 tag 找到对应领域的知识

**示例**：
```
领域：AI
    ↓
查找所有 tag 包含 #AI 的文件
    ↓
统计：待阅读 X 个，已阅读 X 个，已理解 X 个，已掌握 X 个
```

**实现代码**：
```typescript
function getDomainStats(app: App, domain: string): Record<string, number> {
  const stats = {
    待阅读: 0,
    已阅读: 0,
    已理解: 0,
    已掌握: 0,
  };

  app.vault.getMarkdownFiles().forEach(file => {
    const cache = app.metadataCache.getFileCache(file);
    if (cache?.frontmatter?.tags?.includes(domain)) {
      const status = cache.frontmatter.学习状态;
      if (status && status in stats) {
        stats[status]++;
      }
    }
  });

  return stats;
}
```
