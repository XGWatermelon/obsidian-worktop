## 插件简介

**智能工作台插件**是一个 Obsidian 插件，旨在提供一个统一的工作台界面，帮助你管理日记、任务、学习计划和知识笔记。

### 核心功能

- **快速创建**：一键创建任务、想法、项目、灵感、写作、学习、日志
- **工作流管理**：每日开始、任务总览、待整理、任务评估、写作管理、学习统计、最近动态
- **状态监控**：今日待办、本周待办、逾期事项统计
- **学习分析**：热力图、学习状态分布、SAP 模块覆盖

## 安装与配置

### 安装步骤

1. 将以下文件复制到 Obsidian 插件目录：
   ```
   .obsidian/plugins/worktop/
   ├── main.js
   ├── manifest.json
   └── styles.css
   ```

2. 在 Obsidian 中启用插件：
   - 打开设置 → 第三方插件
   - 找到 "Worktop" 并启用

### 配置选项

在设置 → 第三方插件 → Worktop → 点击齿轮图标打开配置页面：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 在主编辑区打开 | 开启 | 开启后，工作台将在主编辑区以标签页形式打开；关闭则在侧边栏打开 |
| 主题颜色 | 暗夜绿 | 选择工作台的主题颜色（10种主题可选） |
| 日记文件夹 | `日记` | 日记文件存放的文件夹路径 |
| 日计划文件夹 | `日计划` | 日计划文件存放的文件夹路径 |
| 周计划文件夹 | `周计划` | 周计划文件存放的文件夹路径 |
| 事项文件夹 | `事项` | 任务、想法、项目、灵感、写作、学习等事项存放的文件夹路径 |
| 待整理文件夹 | `待整理` | 待整理文件存放的文件夹路径 |

**注意**：修改路径设置后，需要重新加载插件才能生效。

```

│  工作流按钮（每日开始 | 任务总览 | 待整理 | 任务评估 | 写作管理 | 学习统计 | 最近动态）│
├─────────────────────────────────────────────────────────────────────────────────┤
```

### 实时刷新机制

工作台支持实时刷新，当文件发生以下变化时会自动刷新：
- 文件创建
- 文件修改
- 文件删除
- 文件重命名

刷新延迟：500ms（防抖机制，避免频繁刷新）

手动刷新：点击标签页导航栏的"刷新"按钮

### 工作流详解

#### 任务/想法/项目/灵感/写作/学习

**触发方式**：点击对应按钮

**工作流程**：
```
点击按钮
    ↓
弹出创建弹窗
    ↓
输入标题
    ↓
生成文件路径：事项/YYYY-MM-DD-标题.md
    ↓
创建文件并填充模板内容
    ↓
打开文件供编辑
```

**模板内容**：
```markdown
---
tags:
  - 事项
type: 任务/想法/项目/灵感/写作/学习
status: 待评估
created: YYYY-MM-DD
updated: YYYY-MM-DD
deadline: ""
学习状态: 待阅读
---

# 标题

## 背景


## 目标


## 进展记录

### YYYY-MM-DD
- 创建事项

## 参考资料

```

### 工作流详解

#### 每日开始

**功能**：开始新的一天，创建或打开今日日记

**触发方式**：点击"每日开始"按钮

**工作流程**：
```
点击按钮
    ↓
获取今日日期：YYYY-MM-DD
    ↓
检查 日记/YYYY-MM-DD.md 是否存在
    ↓ (不存在)
创建日记文件，填充模板内容
    ↓ (已存在)
直接打开现有日记
    ↓
在编辑器中打开日记文件
```

**使用场景**：
- 每天早上开始工作时点击
- 快速创建今日日记，记录今日计划

---

#### 任务总览

**功能**：查看所有进行中的任务

**触发方式**：点击"任务总览"按钮

**工作流程**：
```
点击按钮
    ↓
查询所有 status=进行中的事项
    ↓ (有结果)
打开第一个进行中的事项文件
    ↓ (无结果)
提示"暂无进行中的任务"
```

**使用场景**：
- 快速查看当前正在进行的任务
- 跳转到最近修改的进行中任务

---

#### 待整理清单

**功能**：查看待整理文件夹中的文件

**触发方式**：点击"待整理清单"按钮

**工作流程**：
```
点击按钮
    ↓
检查 待整理/ 文件夹中的文件
    ↓ (有文件)
打开第一个待整理文件
    ↓ (无文件)
提示"待整理文件夹为空"
```

**使用场景**：
- 整理临时笔记
- 处理待归档文件

---

#### 任务评估

**功能**：评估待定的任务

**触发方式**：点击"任务评估"按钮

**工作流程**：

## 状态栏卡片

### 卡片列表

| 卡片 | 功能 | 点击行为 |
|------|------|----------|
| 今日日记 | 打开今日日记 | 跳转到今日日记文件 |
| 今日待办 | 统计今日待办任务 | 跳转到今日日记 |
| 本周待办 | 统计本周待办任务 | 跳转到本周计划 |
| 逾期 | 统计逾期事项 | 跳转到第一个逾期事项 |

### 工作流详解

#### 今日日记

**功能**：显示今日日期，点击打开日记

**数据来源**：系统日期

**点击行为**：
```
点击卡片
    ↓
获取今日日记路径：日记/YYYY-MM-DD.md
    ↓
检查文件是否存在
    ↓ (存在)
打开日记文件
    ↓ (不存在)
创建日记文件并打开
```

---

#### 今日待办

**功能**：统计今日待办任务数量

**数据来源**：今日日记中的 `## 今日要事` 段落

**统计逻辑**：
```
读取今日日记文件
    ↓
查找 ## 今日要事 段落
    ↓
统计 `- [ ]` 数量（未完成）
统计 `- [x]` 数量（已完成）
    ↓
显示格式：已完成/总数
```

**点击行为**：跳转到今日日记

---

#### 本周待办

**功能**：统计本周待办任务数量

**数据来源**：本周7天的日记文件

**统计逻辑**：
```
获取本周日期范围（周一到周日）
    ↓
遍历7天的日记文件
    ↓
统计所有 `- [ ]` 和 `- [x]` 数量
    ↓
显示格式：已完成/总数
```

**点击行为**：跳转到本周计划

---

#### 逾期

**功能**：统计逾期事项数量

**数据来源**：所有事项文件的 deadline 字段

**统计逻辑**：
```
遍历所有事项文件
    ↓
检查 frontmatter.deadline 字段
    ↓
筛选 deadline < 今日 且 status ≠ 已完成/已放弃
    ↓
统计数量
```

**点击行为**：跳转到第一个逾期事项

## 常见问题

### Q1: 工作流按钮点击无效果？

**A**: 检查是否已创建对应的文件：
- 每日开始：需要日记文件夹存在
- 任务总览：需要有 status=进行中的事项
- 待整理清单：需要待整理文件夹有文件
- 任务评估：需要有 status=待评估的事项
- 写作管理：需要有 status=进行中 且 type=写作的事项

### Q2: 状态栏显示"..."？

**A**: 这表示没有找到对应的数据：
- 今日待办：检查今日日记是否有 `## 今日要事` 段落
- 本周待办：检查本周是否有日记文件
- 逾期：检查是否有 deadline 且已过期的事项

### Q3: 如何修改事项类型？

**A**: 直接编辑事项文件的 frontmatter：
```yaml
type: 任务  # 改为：想法、项目、灵感、写作、学习
```

### Q4: 如何修改事项状态？

**A**: 直接编辑事项文件的 frontmatter：
```yaml
status: 进行中  # 改为：待评估、已完成、已放弃
```

### Q5: 热力图不显示？

**A**: 热力图需要至少一年的文件数据。如果是新仓库，热力图会显示为空白。

### Q6: 如何备份配置？

**A**: 配置保存在 `.obsidian/plugins/worktop/data.json` 文件中，备份此文件即可。
