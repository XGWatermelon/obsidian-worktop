# Obsidian 智能工作台插件 — 新用户上手指南

> **版本**：v2.0
> **更新日期**：2026-05-29
> **适用对象**：首次使用本插件的用户

---

## 一、快速开始（5 分钟）

### 步骤 1：安装插件

将以下文件复制到你的 Obsidian 仓库：

```
<你的仓库>/.obsidian/plugins/worktop/
├── main.js
├── manifest.json
└── styles.css
```

### 步骤 2：启用插件

1. 打开 Obsidian
2. 设置 → 第三方插件 → 找到「智能工作台」
3. 开启插件

### 步骤 3：打开工作台

- 点击左侧边栏的 📊 图标
- 或按 `Ctrl+P` 输入「打开工作台」

**首次打开时，插件会自动创建所需的文件夹结构。**

## 二、目录结构

### 2.1 推荐的目录结构

```
你的 Obsidian 仓库/
│
├── 工作台/                          ← 工作台根目录（可自定义名称）
│   ├── 日计划/                     ← 自动生成
│   │   ├── 2026-05-29-日计划.md
│   │   └── ...
│   │
│   ├── 周计划/                     ← 自动生成
│   │   ├── 2026-W22-本周计划.md
│   │   └── ...
│   │
│   ├── 事项/                       ← 用户创建
│   │   ├── 2026-05-29-学习TypeScript.md
│   │   └── ...
│   │
│   └── 常用入口.md                 ← 自动生成/用户编辑
│
├── 节点/                            ← 知识图谱节点（与工作台平级）
│   ├── AI/
│   │   ├── MCP协议.md
│   │   └── LangChain框架.md
│   └── SAP/
│       └── ...
│
├── 待整理/                          ← 收件箱
│   ├── 收藏摘录/
│   └── 待归类/
│
└── .obsidian/
    └── plugins/
        └── worktop/
            ├── main.js
            ├── manifest.json
            ├── styles.css
            └── config.json         ← 配置文件
```

### 2.2 为什么节点与工作台平级？

| 文件夹 | 性质 | 说明 |
|--------|------|------|
| 工作台/ | 任务管理 | 日计划、周计划、事项 — 有时间属性 |
| 节点/ | 知识积累 | 知识图谱节点 — 无时间属性，永久存在 |
| 待整理/ | 临时存储 | 收件箱，待归类内容 |

**节点的作用**：
- Obsidian 图谱视图的基础单元
- Wiki 链接 `[[节点/MCP协议]]` 的目标
- 帮助 AI 理解知识结构

## 三、配置说明

### 3.1 配置文件位置

```
.obsidian/plugins/worktop/config.json
```

### 3.2 配置结构

配置分为 **基础配置**（日常使用）和 **高级配置**（一般不需要改）：

```json
{
  "version": 1,
  "basic": {
    "app": { "name": "工作台", "icon": "layout-dashboard", "viewTitle": "智能工作台" },
    "ui": { "openInMainEditor": true, "themeId": "green-dark" },
    "folders": { "root": "工作台", "structure": { ... } },
    "domains": [ ... ],
    "statuses": { "topic": [...], "learning": [...], "default": "...", "completed": [...] },
    "types": [ ... ],
    "workflows": [ ... ]
  },
  "advanced": {
    "tabs": [ ... ],
    "fields": { ... },
    "limits": { ... },
    "theme": { ... },
    "templates": { ... }
  }
}
```

### 3.3 通过设置界面配置

1. 打开 Obsidian 设置（快捷键：`,`）
2. 第三方插件 → 智能工作台 → 点击齿轮图标 ⚙️
3. 修改配置项并保存

**设置界面分为两层**：
- **基本配置**：打开方式、主题、文件夹、状态、类型、领域
- **高级配置**：标签页、字段映射、数值限制、颜色、模板

## 四、核心功能

### 4.1 创建日计划

**触发方式**：
- 命令：`Ctrl+P` → 输入「创建今日日计划」
- 工作台：点击「每日开始」按钮

**自动行为**：
1. 创建文件夹：`工作台/日计划/`（如果不存在）
2. 创建文件：`工作台/日计划/2026-05-29-日计划.md`
3. 如果周计划不存在，会询问是否创建
4. 自动关联周计划（在周计划中添加链接）

**生成的文件内容**：
```markdown
---
tags:
  - 计划
  - 日计划
type: 日计划
date: 2026-05-29
weekly_plan: "[[工作台/周计划/2026-W22-本周计划.md]]"
---

# 2026-05-29 日计划

> 本周计划：[[工作台/周计划/2026-W22-本周计划.md]]

## 今日要事

- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3

## 执行记录

### 上午

-

### 下午

-

### 晚上

-

## 完成情况

- 完成：
- 未完成：
- 原因：
```

### 4.3 创建事项

**触发方式**：
- 命令：`Ctrl+P` → 输入「创建事项」

**弹窗内容**：
- 标题（必填）
- 状态（下拉选择）
- 学习状态（下拉选择）

## 六、配置项详解

### 6.1 基础配置

#### 应用配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.app.name` | `工作台` | 应用名称 |
| `basic.app.icon` | `layout-dashboard` | 侧边栏图标 |
| `basic.app.viewTitle` | `智能工作台` | 视图标题 |

#### UI 配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.ui.openInMainEditor` | `true` | 是否在主编辑区打开 |
| `basic.ui.themeId` | `green-dark` | 主题 ID |

#### 文件夹配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.folders.root` | `工作台` | 根目录名称 |
| `basic.folders.structure.dailyPlan.name` | `日计划` | 日计划文件夹 |
| `basic.folders.structure.dailyPlan.pattern` | `{date}-日计划` | 日计划文件名模式 |
| `basic.folders.structure.weeklyPlan.name` | `周计划` | 周计划文件夹 |
| `basic.folders.structure.weeklyPlan.pattern` | `{year}-W{week}-本周计划` | 周计划文件名模式 |
| `basic.folders.structure.topic.name` | `事项` | 事项文件夹 |
| `basic.folders.structure.nodes.name` | `节点` | 节点文件夹 |
| `basic.folders.structure.inbox.name` | `待整理` | 待整理文件夹 |

**文件名模式变量**：
- `{date}` — 日期，如 `2026-05-29`
- `{year}` — 年份，如 `2026`
- `{week}` — 周数，如 `22`
- `{month}` — 月份，如 `05`
- `{day}` — 日期，如 `29`

#### 状态配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.statuses.topic` | `["待评估", "进行中", "已完成", "已放弃"]` | 事项状态列表 |
| `basic.statuses.learning` | `["待阅读", "已阅读", "已理解", "已掌握"]` | 学习状态列表 |
| `basic.statuses.default` | `待评估` | 新建事项的默认状态 |
| `basic.statuses.completed` | `["已完成", "已放弃"]` | 视为「已完成」的状态 |

#### 类型配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.types` | `[{id:"task",name:"任务"}, ...]` | 事项类型列表 |

**默认类型**：
- `task` — 任务
- `idea` — 想法
- `project` — 项目
- `writing` — 写作
- `learning` — 学习

#### 领域配置

```json
{
  "basic.domains": [

      "id": "ai",
```

|--------|--------|------|

**默认工作流**：
- `daily-start` — 每日开始

- `task-eval` — 任务评估

## 八、常见问题

### Q1: 首次打开工作台后，文件夹是怎么创建的？

**答**：插件采用「按需创建」策略：
- 创建日计划时，自动创建 `日计划/` 文件夹
- 创建周计划时，自动创建 `周计划/` 文件夹
- 创建事项时，自动创建 `事项/` 文件夹

**不会**在插件启动时批量创建所有文件夹。

### Q2: 为什么节点文件夹与工作台平级？

**答**：
- **工作台**（日计划、周计划、事项）是「时间驱动」的任务管理
- **节点**是「知识驱动」的知识积累，用于 Obsidian 图谱视图和 Wiki 链接
- 两者性质不同，平级更清晰

### Q3: 如何自定义周计划的分类？

**答**：修改配置中的 `advanced.templates.weeklyCategories`：

```json
{
  "advanced": {
    "templates": {
      "weeklyCategories": ["我的分类1", "我的分类2", "我的分类3"]
    }
  }
}
```

### Q4: 如何修改事项状态列表？

**答**：修改配置中的 `basic.statuses.topic`：

```json
{
  "basic": {
    "statuses": {
      "topic": ["待办", "进行中", "完成", "取消"],
      "default": "待办",
      "completed": ["完成", "取消"]
    }
  }
}
```

**注意**：修改后需要同步更新 `basic.statuses.default` 和 `basic.statuses.completed`。

### Q5: 如何添加新的事项类型？

**答**：修改配置中的 `basic.types`：

```json
{
  "basic": {
    "types": [
      { "id": "task", "name": "任务" },
      { "id": "bug", "name": "Bug" },
      { "id": "feature", "name": "需求" }
    ]
  }
}
```

### Q6: 如何添加新的领域和模块？

**答**：修改配置中的 `basic.domains`：

```json
{
  "basic": {
    "domains": [
      {
        "id": "frontend",
        "name": "前端",
        "color": "#3B82F6",
        "modules": [
          { "id": "react", "name": "React" },
          { "id": "vue", "name": "Vue" }
        ]
      }
    ]
  }
}
```

### Q7: 如何备份配置？

**答**：复制 `.obsidian/plugins/worktop/config.json` 文件即可。

### Q8: 配置文件在哪里？

**答**：`.obsidian/plugins/worktop/config.json`
