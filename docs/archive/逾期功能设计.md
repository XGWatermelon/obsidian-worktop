## 逾期分类

### 1. 事项逾期

**来源**：事项文件夹（事项/*.md）

**判断条件**：
- deadline < 今日日期
- status ≠ "已完成"
- status ≠ "已放弃"

**示例**：
```markdown
事项/2026-05-20-学习FICO.md
---
status: 进行中
deadline: 2026-05-25
---
→ 逾期3天
```

### 2. 计划逾期

**来源**：
- 周计划文件（日记/*-本周计划.md）
- 日记文件（日记/*.md）

**判断条件**：
```
周计划逾期：
- 周计划的 end_date < 今日日期
- 周计划中的任务未完成（- [ ]）

日记逾期：
- 日记日期 < 今日日期
- 日记中的任务未完成（- [ ]）
```

## 逾期页面设计

### 页面结构

```
逾期页面
├── 逾期统计卡片
│   ├── 事项逾期：3个
│   ├── 计划逾期：2个
│   └── 总计：5个
│
├── 事项逾期列表
│   ├── 任务1（逾期3天）
│   ├── 任务2（逾期1天）
│   └── 任务3（逾期5天）
│
└── 计划逾期列表
    ├── 周计划1（逾期2天）
    └── 日记1（逾期2天）
```

### 操作按钮

- [打开] - 打开对应文件
- [标记完成] - 将任务标记为已完成
- [延期] - 修改截止日期
- [全部标记完成] - 批量标记所有逾期任务
