## 架构概览

### 核心设计理念

**工作台 = 展示层**，**主数据 = 文件层**

```
┌─────────────────────────────────────────────────────────────────┐
│                        工作台插件（展示层）                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │ 行动指北  │  │ 知识管理  │  │ 学习分析  │  │ 稍后阅读  │       │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │
│                           ↓ 读取/跳转                           │
├─────────────────────────────────────────────────────────────────┤
│                        主数据（文件层）                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │  日计划   │  │  周计划   │  │   事项   │  │  日记    │       │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

### 功能分类

| 功能类型 | 说明 | 示例 |
|----------|------|------|
| **统计** | 读取主数据，计算数量 | 今日待办数、本周完成数、逾期数 |
| **展示** | 生成临时文档，展示详细列表 | 逾期事项列表、已完成列表、文档概览 |
| **跳转** | 点击后打开主数据文件 | 点击事项跳转到事项文件 |
| **创建** | 创建新的主数据文件 | 创建日计划、创建事项 |

## 一、主数据结构（文件层）

### 1.1 日计划

**路径**：`日计划/YYYY-MM-DD-日计划.md`

**作用**：记录今天要做什么，是执行层

**模板**：
```markdown
---
tags:
  - 计划
  - 日计划
type: 日计划
date: 2026-05-28
weekly_plan: "[[周计划/2026-W22-本周计划.md]]"
---

# 2026-05-28 日计划

> 本周计划：[[周计划/2026-W22-本周计划.md]]

## 今日要事

- [ ] 任务 1
- [ ] 任务 2
```

### 1.2 周计划

**路径**：`周计划/YYYY-WXX-本周计划.md`

**作用**：记录本周要做什么，是规划层

**模板**：
```markdown
---
tags:
  - 计划
  - 周计划
type: 周计划
week: "2026-W22"
start_date: 2026-05-25
end_date: 2026-05-31
status: 进行中
---

# 2026 年第 22 周计划

## 本周目标

### SAP 学习
- [ ] 学习内容 1

### 项目任务
- [ ] 任务 1

## 每日进展

### 星期一 - 05.25
- [[日计划/2026-05-25-日计划.md]]

### 星期二 - 05.26
- [[日计划/2026-05-26-日计划.md]]
```

### 1.3 事项

**路径**：`事项/YYYY-MM-DD-标题.md`

**作用**：记录长期任务、想法、项目等

**类型**：任务、想法、项目、灵感、写作、学习

**状态**：待评估、进行中、已完成、已放弃

**模板**：
```markdown
---
tags:
  - 事项
type: 任务
status: 进行中
created: 2026-05-28
updated: 2026-05-28
deadline: "2026-06-01"
学习状态: 待阅读
---

# 标题

## 背景

## 目标

## 进展记录

### 2026-05-28
- 创建事项
```

### 1.4 日记

**路径**：`日记/YYYY-MM-DD.md`

**作用**：记录今天做了什么，是复盘层

**说明**：日记是用户自己维护的，插件不涉及创建和修改

## 二、工作台插件（展示层）

### 2.1 状态栏

**功能**：显示统计数据，点击生成详细列表或跳转

| 卡片 | 数据来源 | 点击行为 |
|------|----------|----------|
| 今日日计划 | 日计划文件是否存在 | 跳转到日计划 |
| 今日待办 | 日计划中的任务数 | 跳转到日计划 |
| 本周待办 | 周计划 + 7天日计划的任务数 | 跳转到周计划 |
| 逾期 | 事项中 deadline < 今日的 | 生成逾期列表文档 |
| 已完成 | 事项中 status=已完成的 | 生成已完成列表文档 |
| 进行中 | 事项中 status=进行中的 | 生成进行中列表文档 |

**生成的文档**（可删除，不影响统计）：
- `工作台/逾期事项.md`
- `工作台/已完成事项.md`
- `工作台/进行中事项.md`

### 2.2 快捷操作栏

**功能**：快速创建和常用操作

| 按钮组 | 功能 |
|--------|------|
| 创建 | 快速创建任务、想法、项目、灵感、写作、学习、日志 |
| 工作流 | 每日开始、任务总览、待整理清单、任务评估、写作管理、学习统计、最近动态 |

### 2.3 行动指北标签页

**功能**：日常工作概览

| 模块 | 功能 |
|------|------|
| 今日要事 | 显示日计划中的任务 |
| 活跃任务 | 显示进行中的事项 |
| 常用入口 | 可配置的常用文档链接（支持添加/删除） |
| 本周计划 | 显示周计划中的目标 |
| 学习进度 | 显示学习状态分布 |
| 最近修改 | 显示最近修改的文件 |

### 2.4 知识管理标签页

**功能**：管理事项和笔记

| 模块 | 功能 |
|------|------|
| 事项清单 | 显示进行中的事项 |
| 事项池 | 显示待评估的事项 |
| 最近笔记 | 显示最近修改的笔记 |
| 文档概览 | 按文件夹统计文档数量，点击生成详细文档 |

**生成的文档**（可删除，不影响统计）：
- `工作台/文档概览.md`
- `工作台/XXX文档清单.md`

### 2.5 学习分析标签页

**功能**：学习数据可视化

| 模块 | 功能 |
|------|------|
| 热力图 | 显示过去一年的活跃度 |
| 学习状态分布 | 待阅读/已阅读/已理解/已掌握 |
| SAP 模块覆盖 | FICO/HR/MM/SD 学习进度 |

### 2.6 稍后阅读标签页

**功能**：管理待阅读内容

| 模块 | 功能 |
|------|------|
| 收藏摘录 | 显示收藏的笔记 |
| 待归类 | 显示待整理的笔记 |

## 三、数据流向

### 3.1 读取数据

```
工作台插件
    ↓ 调用 Obsidian API
    ↓ app.vault.getMarkdownFiles()
    ↓ app.metadataCache.getFileCache()
主数据文件
```

### 3.2 创建数据

```
用户点击"创建日计划"
    ↓ 检查周计划是否存在
    ↓ 如果不存在，提示创建周计划
    ↓ 创建日计划文件
    ↓ 在周计划中添加日计划链接
主数据文件
```

### 3.3 生成文档

```
用户点击"逾期"
    ↓ 读取事项文件夹
    ↓ 筛选逾期事项
    ↓ 生成 Markdown 内容
    ↓ 创建/更新 工作台/逾期事项.md
    ↓ 打开文档
生成的文档（可删除，不影响统计）
```

## 四、实时刷新机制

### 4.1 自动刷新

**触发条件**：
- 文件创建
- 文件修改
- 文件删除
- 文件重命名

**防抖机制**：500ms 延迟，避免频繁刷新

### 4.2 手动刷新

**方式**：切换标签页时自动刷新

## 八、关键设计决策

### 8.1 工作台 vs 主页

- **工作台**：动态展示，统计+跳转+创建
- **主页**：静态展示，全量文档清单

### 8.2 生成的文档 vs 主数据

- **生成的文档**：可删除，不影响统计，每次点击重新生成
- **主数据**：不可随意删除，修改会影响统计

### 8.3 实时刷新 vs 手动刷新

- **实时刷新**：文件变化时自动刷新（500ms 防抖）
- **手动刷新**：切换标签页时刷新
