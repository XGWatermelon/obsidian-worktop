# 设计报告版本控制说明

## 文件命名规范

```
design-review-v{版本号}-{日期}.md
```

示例：
- `design-review-v1.0-2026-05-29.md` — 初始版本
- `design-review-v1.1-2026-06-01.md` — 修订版本
- `design-review-v2.0-2026-06-15.md` — 重大重构版本

## 版本号规则

- **主版本号 (x.0)** — 架构重大变更、推翻重来
- **次版本号 (x.y)** — 新增分析维度、补充发现
- **修订号** — 修正错误、补充细节（不单独发文件，在文件内记录）

## 变更记录格式

每个报告文件末尾包含变更历史：

```markdown
## 变更历史

| 版本 | 日期 | 变更内容 |
|------|------|----------|
| v1.0 | 2026-05-29 | 初始版本，全面审查 |
| v1.1 | 2026-06-01 | 补充性能分析 |
```
