- SAP ABAP 开发者，有 Java/Python 基础
- 目前在学习 SAP 业务知识（已学 FICO 部分）
- 参加过几个 SAP 二次开发项目，积攒了经验和文章
- 目标：结构化个人知识库

### 现有知识库

- **笔记库**：`F:\personal_files\个人知识库整理\Obsidian-warehouse\obsidian-note-warehouse`
- **Wiki 库**：`F:\personal_files\个人知识库整理\Obsidian-warehouse\obsidian-wiki-knowledge-base`（Karpathy 风格，测试中）
- 笔记库现有 126 个文件，12 个顶层文件夹

## 三、选题功能设计（已确认）

### 选题定义

选题 = 你想做的事情，包括写作选题、学习选题、项目灵感等。

### 选题类型

| 类型 | 说明 | 举例 |
|------|------|------|
| `写作` | 要写的文章/分享 | "SAP FICO 配置踩坑总结" |
| `学习` | 要深入研究的知识点 | "成本中心会计的配置逻辑" |
| `项目` | 要做的工具/实验 | "做一个 ABAP 代码生成器" |
| `灵感` | 未成形的闪念 | "SAP 和 AI 结合的可能性" |
| `任务` | 具体待办事项 | "整理 Mizkan 项目文档" |
| `想法` | 随手记录的念头 | "写一篇 Obsidian 使用心得" |

### 选题生命周期

```
创建 → 待评估 → 进行中 → 已完成
                  └→ 已放弃
```

### 选题文件结构

- **存放位置**：`选题/` 文件夹
- **文件命名**：`YYYY-MM-DD-选题标题.md`
- **Frontmatter 字段**：
  - `type`: 写作/学习/项目/灵感/任务/想法
  - `status`: 待评估/进行中/已完成/已放弃
  - `created`: 创建日期
  - `updated`: 更新日期
  - `deadline`: 截止日期（可选，用于逾期判断）
  - `tags`: 标签

### 选题面板展示

在主页"选题清单"面板中，用 Dataview 查询展示：
- 标题
- 类型
- 状态
- 创建时间
- 按状态分组，进行中排最前面

## 四、主页插件设计（已确认方案）

### 实现方案

**方案 C：Dataview + Templater + 插件壳**

- 插件负责 UI 框架和交互（标签页、弹窗、按钮）
- Dataview 负责数据查询
- Templater 负责模板生成
- CSS 负责视觉样式

### 整体布局

```
┌─────────────────────────────────────────────────┐
│  顶部状态栏（4 个数据卡片）                        │
│  [今日日志] [今日待办] [本周待办] [逾期]           │
├─────────────────────────────────────────────────┤
│  快捷操作栏                                       │
│  [任务][想法][项目][灵感][写作][学习][日志] | [每日开始]...    │
├─────────────────────────────────────────────────┤
│  标签页导航                                       │
│  [行动指北] [知识管理] [学习分析] [稍后阅读]       │
├─────────────────────────────────────────────────┤
│                                                 │
│  内容区（根据选中标签页切换）                       │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 五、顶部状态栏设计

### 4 个数据卡片

| 卡片 | 数据来源 | 统计方式 | 点击行为 |
|------|----------|----------|----------|
| 今日日志 | `日记/YYYY-MM-DD.md` | 文件是否存在 | 存在→打开日记；不存在→显示"未创建"，点击跳转创建页 |
| 今日待办 | 今日日记中的 `- [ ]` | Dataview 统计 | 跳转今日日记 |
| 本周待办 | 本周所有日记中的任务 | Dataview 统计 | 打开本周任务视图 |
| 逾期 | 超过 `deadline` 的未完成任务 | Dataview 统计 | 打开逾期任务视图 |

### 视觉样式

- 圆角卡片，轻微阴影
- 不同状态用不同颜色：
  - 绿色（#77DD77）：正常/已完成
  - 淡紫色（#CBC3E3）：本周待办
  - 深灰（#333333）：逾期/无数据
- 点击可跳转

## 六、快捷操作栏设计

### 快速创建按钮（7 个）

| 按钮 | 保存位置 | 文件名格式 | 模板 |
|------|----------|------------|------|
| 任务 | `选题/` | `YYYY-MM-DD-任务标题.md` | type=任务, status=待评估 |
| 想法 | `选题/` | `YYYY-MM-DD-想法标题.md` | type=想法, status=待评估 |
| 项目 | `选题/` | `YYYY-MM-DD-项目标题.md` | type=项目, status=待评估 |
| 灵感 | `选题/` | `YYYY-MM-DD-灵感标题.md` | type=灵感, status=待评估 |
| 写作 | `选题/` | `YYYY-MM-DD-写作标题.md` | type=写作, status=待评估 |
| 学习 | `选题/` | `YYYY-MM-DD-学习标题.md` | type=学习, status=待评估 |
| 日志 | `日记/` | `YYYY-MM-DD.md` | 日记模板 |

**交互流程**：
1. 点击按钮 → 弹出输入框
2. 输入标题（可选输入描述）
3. 自动创建文件，填充 frontmatter 和模板

### 工作流按钮（8 个）

| 按钮 | 功能 | 实现方式 |
|------|------|----------|
| 每日开始 | 打开今日日记 + 插入模板 + 显示今日待办和逾期任务 | Templater 模板 + Dataview 查询 |
| 任务总览 | 查看所有进行中选题 + 批量修改状态 + 查看逾期任务 | Dataview 视图 + 交互面板 |
| 待整理清单 | 显示待整理文件列表，点击可跳转 | Dataview 视图 |
| 文件归档 | 把待整理文件移到正式文件夹 | 文件移动操作 |
| 任务评估 | 查看待评估任务，决定做/不做，批量改状态 | Dataview 视图 + 交互面板 |
| 写作管理 | 打开正在写的笔记 | Dataview 视图 |
| 学习统计 | 查看学习状态分布 | Dataview 视图 |
| 最近动态 | 查看最近新增/更新的笔记 | Dataview 视图 |

### 视觉样式

- 圆角矩形，深色背景
- hover 高亮效果
- 两组之间用分隔线区分

## 七、标签页设计

### 标签页 1：行动指北

**左侧主内容区：**

| 卡片 | 数据来源 | 内容 |
|------|----------|------|
| 今日要事 | `日记/YYYY-MM-DD.md` | 今日核心任务列表；不存在时显示"未创建"，点击跳转创建页 |
| 活跃任务 | `选题/` (status=进行中) | 进行中的选题列表，带类型标签 |

**右侧信息面板：**

| 卡片 | 数据来源 | 内容 |
|------|----------|------|
| 本周计划 | `日记/YYYY-WXX-本周计划.md` | 本周任务列表；不存在时显示"未创建"，点击跳转创建页 |
| 学习进度 | 全库 `学习状态` 字段 | 待阅读/已阅读/已理解/已掌握 统计（所有笔记均可设置，部分可能缺失） |
| 最近修改 | 全库文件 `file.mtime` | 最近 7 天修改的笔记列表 |

### 标签页 2：知识管理

| 区域 | 数据来源 | 内容 |
|------|----------|------|
| 选题清单 | `选题/` (status=进行中) | 进行中的选题，按类型分组 |
| 选题池 | `选题/` (status=待评估) | 待评估的选题 |
| 最近笔记 | 全库最近修改的笔记 | Dataview 查询最近 7 天 |

### 标签页 3：学习分析

| 区域 | 数据来源 | 内容 |
|------|----------|------|
| 年度活跃度热力图 | 全库文件创建/修改时间 | GitHub 风格热力图，显示最近一年每日活动量 |
| 学习状态分布 | 全库 `学习状态` 字段 | 待阅读/已阅读/已理解/已掌握 各多少篇（所有笔记均可设置，部分可能缺失） |
| 模块覆盖 | `SAP/模块/` | 各模块学习状态细分 |
| 对象图谱 | `对象/` | 知识图谱节点统计 |

#### 年度活跃度热力图设计

**布局**：放在"学习分析"标签页顶部，宽度占满

**样式**：
- GitHub 个人主页风格：按周排列，以月为列分组
- 卡片式圆角设计，标题"年度活跃度热力图"
- 鼠标 hover 显示当天具体笔记数量

**颜色规则**：
- 0 活动：浅色（深色模式 #161b22，浅色模式 #ebedf0）
- 活动越多颜色越饱和：深绿/蓝渐变
- 深色模式：#0e4429 → #006d32 → #26a641 → #39d353
- 浅色模式：#9be9a8 → #40c463 → #30a14e → #216e39

**数据来源**：
- 自动统计 Obsidian 中当天创建/修改的笔记数量
- 无需手动维护
- 实现方式：DataviewJS 查询全库文件的 `file.ctime` 和 `file.mtime`

**实现方案**：
- 在自定义插件中用 DataviewJS 查询数据
- 用 HTML/CSS 渲染热力图网格
- 或使用现有插件 `obsidian-heatmap-calendar` 作为依赖

### 标签页 4：稍后阅读

| 区域 | 数据来源 | 内容 |
|------|----------|------|
| 收藏摘录 | `待整理/收藏摘录/` | 收藏但未读的文章 |
| 待归类 | `待整理/待归类/` | 已成型但未归类的笔记 |

### 颜色方案

- 辅助色（紫）：#CBC3E3

- 辅助色（紫）：#9C27B0

- 标题：无衬线粗体
- 正文：轻体

- 读取 Obsidian 主题设置（`app.getTheme()`）

## 九、技术实现方案

### 依赖插件

| 插件 | 用途 | 必需？ |
|------|------|--------|
| Dataview | 数据查询和统计 | 是 |
| Templater | 模板生成 | 是 |
| QuickAdd | 快捷创建笔记 | 可选（插件可自己实现） |

### 插件架构

```
obsidian-workspace-plugin/
├── main.ts              # 插件入口
├── views/
│   ├── WorkspaceView.ts # 主视图（标签页容器）
│   ├── StatusBar.ts     # 顶部状态栏
│   ├── QuickActions.ts  # 快捷操作栏
│   └── tabs/
│       ├── ActionTab.ts # 行动指北
│       ├── KnowledgeTab.ts # 知识管理
│       ├── AnalysisTab.ts # 学习分析
│       └── ReadingTab.ts # 稍后阅读
├── utils/
│   ├── dataview.ts      # Dataview 查询封装
│   ├── templater.ts     # Templater 调用封装
│   └── theme.ts         # 主题切换逻辑
├── styles/
│   ├── main.css         # 主样式
│   └── themes.css       # 主题变量
└── manifest.json
```

### 数据流

```
用户点击按钮
    ↓
插件捕获事件
    ↓
调用 Obsidian API / Dataview / Templater
    ↓
读取/创建笔记文件
    ↓
更新 UI 显示
```
