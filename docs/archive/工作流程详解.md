# 工作流程详解

> 本文档详细说明插件的运行机制，帮助理解「哪些内容是自动生成的」

---

## 一、插件启动流程

当 Obsidian 启动并加载插件时，按以下顺序执行：

```
1. 加载配置
   ├── 读取 data.json（Obsidian 自动管理）
   ├── 读取 config.json（插件目录下，可选）
   └── 合并默认配置 → 生成最终配置

2. 注册组件
   ├── 注册设置页面（设置界面）
   ├── 注册视图（工作台视图）
   ├── 注册侧边栏图标
   └── 注册命令（4个命令）

3. 等待用户操作
```

**关键点**：启动时不会自动创建任何文件或文件夹！

## 二、用户操作触发的流程

### 2.1 打开工作台

**触发方式**：点击侧边栏图标 / 命令「打开工作台」

```
用户点击「打开工作台」
    ↓
检查是否已有打开的工作台视图
    ├── 是 → 切换到已有视图
    └── 否 → 创建新视图
            ↓
        渲染工作台界面
            ├── 状态栏（今日日计划、本周待办、逾期、已完成、进行中）
            ├── 快捷操作栏（创建按钮、工作流按钮）
            ├── 标签页导航
            └── 标签页内容
```

**此时会读取的数据**：
- 扫描 vault 中的 markdown 文件
- 读取 frontmatter 中的 status、type、deadline 等字段
- 统计各状态的事项数量

**不会自动创建任何内容**。

---

### 2.2 创建日计划

**触发方式**：命令「创建今日日计划」/ 点击「每日开始」按钮

```
用户触发「创建日计划」
    ↓
获取日计划路径：{workspaceRoot}/日计划/{date}-日计划.md
    ↓
检查日计划文件夹是否存在
    ├── 否 → 自动创建文件夹 ⚠️
    └── 是 → 继续
    ↓
检查今日日计划是否已存在
    ├── 是 → 直接打开
    └── 否 → 检查周计划是否存在
            ├── 否 → 弹窗询问「是否先创建本周计划？」
            │       ├── 确认 → 创建周计划 → 创建日计划 → 关联
            │       └── 取消 → 不创建
            └── 是 → 创建日计划 → 关联到周计划
```

**自动生成的内容**：

1. **文件夹**：`{workspaceRoot}/日计划/`（如果不存在）
2. **日计划文件**：
```markdown
---
tags:
  - 计划
  - 日计划
type: 日计划
date: 2026-05-29
weekly_plan: "[[工作台/周计划/2026-W22-本周计划.md]]"
---

# 2026-05-29 日计划

> 本周计划：[[工作台/周计划/2026-W22-本周计划.md]]

## 今日要事

- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3

## 执行记录

### 上午

-

### 下午

-

### 晚上

-

## 完成情况

- 完成：
- 未完成：
- 原因：
```

3. **周计划中的链接**（自动添加）：
```markdown
### 星期四 - 05.29
- [[工作台/日计划/2026-05-29-日计划.md]]
```

### 2.4 创建事项

**触发方式**：命令「创建事项」/ 点击快捷创建按钮（任务/想法/项目/灵感/写作/学习）

```
用户点击「任务」按钮
    ↓

    ├── 标题（必填）

    ↓
```

### 2.5 生成统计文档

**触发方式**：点击状态栏卡片（逾期/已完成/进行中）

```
用户点击「逾期」卡片
    ↓
扫描所有事项文件
    ↓
筛选出逾期的事项（deadline < 今天 且 status ≠ 已完成/已放弃）
    ↓
生成文档：{workspaceRoot}/逾期事项.md
    ↓
打开文档
```

**自动生成的文档**：
```markdown
# 逾期事项列表

> 生成时间：2026/5/29 14:30:00

### [[工作台/事项/2026-05-20-学习LangChain.md]]
- **类型**：学习
- **状态**：进行中
- **截止日期**：2026-06-01
- **逾期天数**：-3天
- **创建时间**：2026-05-20
```

**类似地，点击「已完成」或「进行中」也会生成对应文档**。

## 三、配置如何影响行为

### 3.1 文件夹路径

| 配置项 | 默认值 | 影响范围 |
|--------|--------|----------|
| `folders.root` | `工作台` | 日计划、周计划、事项的根目录 |
| `folders.structure.dailyPlan.name` | `日计划` | 日计划文件夹名 |
| `folders.structure.weeklyPlan.name` | `周计划` | 周计划文件夹名 |
| `folders.structure.topic.name` | `事项` | 事项文件夹名 |
| `folders.structure.inbox.name` | `待整理` | 待整理文件夹名 |

**路径拼接规则**：
- 日计划路径 = `{root}/{dailyPlan.name}/{pattern}.md`
  - 示例：`工作台/日计划/2026-05-29-日计划.md`
- 事项路径 = `{root}/{topic.name}/{date}-{title}.md`
  - 示例：`工作台/事项/2026-05-29-学习TypeScript.md`

**特殊规则**：如果 `name` 包含 `/`，则视为绝对路径，不拼接 root。
- `savedArticles.name` = `待整理/收藏摘录` → 路径就是 `待整理/收藏摘录`

### 3.2 文件命名模式

| 配置项 | 默认值 | 可用变量 |
|--------|--------|----------|
| `folders.structure.dailyPlan.pattern` | `{date}-日计划` | `{date}`, `{year}`, `{week}` |
| `folders.structure.weeklyPlan.pattern` | `{year}-W{week}-本周计划` | `{date}`, `{year}`, `{week}` |

**示例**：
- `{date}-日计划` → `2026-05-29-日计划`
- `{year}-W{week}-本周计划` → `2026-W22-本周计划`

### 3.3 状态配置

| 配置项 | 默认值 | 影响范围 |
|--------|--------|----------|
| `statuses.topic` | `["待评估", "进行中", "已完成", "已放弃"]` | 事项状态下拉框、统计筛选 |
| `statuses.learning` | `["待阅读", "已阅读", "已理解", "已掌握"]` | 学习状态下拉框、统计筛选 |
| `statuses.default` | `待评估` | 新建事项的默认状态 |

### 3.4 类型配置

| 配置项 | 默认值 | 影响范围 |
|--------|--------|----------|
| `types` | `["任务", "想法", "项目", "灵感", "写作", "学习"]` | 创建按钮、类型下拉框 |

### 3.5 周计划分类

| 配置项 | 默认值 | 影响范围 |
|--------|--------|----------|
| `templates.weeklyCategories` | `["SAP 学习", "项目任务", "写作/事项"]` | 周计划模板中的 `## 本周目标` 分类 |

## 四、「莫名其妙」内容来源追踪

### 4.1 自动创建的文件夹

**触发条件**：首次创建日计划/周计划/事项时

**自动创建的文件夹**：
- `{workspaceRoot}/日计划/`
- `{workspaceRoot}/周计划/`
- `{workspaceRoot}/事项/`

**如何避免**：提前手动创建这些文件夹，插件检测到存在就不会再创建。

### 4.2 自动生成的文件内容

**日计划模板中的默认任务**：
```markdown
## 今日要事

- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3
```

**来源**：`templates.ts` 中的 `getDailyPlanTemplate()` 函数

**周计划模板中的默认分类**：
```markdown
## 本周目标

### SAP 学习

- [ ] 内容 1
- [ ] 内容 2

### 项目任务

- [ ] 内容 1
- [ ] 内容 2

### 写作/事项

- [ ] 内容 1
- [ ] 内容 2
```

**来源**：
- 分类名称来自 `advanced.templates.weeklyCategories` 配置
- 默认内容来自 `templates.ts` 中的 `getWeeklyPlanTemplate()` 函数

### 4.3 自动生成的统计文档

**触发条件**：点击状态栏卡片（逾期/已完成/进行中）、点击领域卡片

**自动生成的文件**：
- `{workspaceRoot}/逾期事项.md`
- `{workspaceRoot}/已完成事项.md`
- `{workspaceRoot}/进行中事项.md`
- `{workspaceRoot}/领域统计-AI.md`
- `{workspaceRoot}/领域统计-SAP.md`
- `{workspaceRoot}/AI文档清单.md`
- `{workspaceRoot}/文档概览.md`

**如何避免**：不点击这些卡片就不会生成。

### 4.4 周计划中自动添加的链接

**触发条件**：创建日计划时，自动在周计划的「每日进展」部分添加链接

**自动添加的内容**：
```markdown
### 星期四 - 05.29
- [[工作台/日计划/2026-05-29-日计划.md]]
```

**来源**：`main.ts` 中的 `addDailyPlanLinkToWeeklyPlan()` 函数

## 六、建议的配置流程

### 6.1 首次使用前

1. **确定你的文件夹结构**
   - 根目录叫什么？（默认：`工作台`）
   - 日计划放哪里？（默认：`工作台/日计划/`）
   - 周计划放哪里？（默认：`工作台/周计划/`）
   - 事项放哪里？（默认：`工作台/事项/`）

2. **修改配置**
   - 设置 → 第三方插件 → 智能工作台 → 齿轮图标
   - 修改「工作台根目录」和各文件夹名称

3. **手动创建文件夹**（可选）
   - 如果不想让插件自动创建，提前手动创建好

### 6.2 配置示例

**示例 1：使用「我的知识库」作为根目录**

配置：
```json
{
  "basic": {
    "folders": {
      "root": "我的知识库",
      "structure": {
        "dailyPlan": { "name": "每日计划", "pattern": "{date}-计划" },
        "weeklyPlan": { "name": "每周计划", "pattern": "W{week}" },
        "topic": { "name": "任务" }
      }
    }
  }
}
```

手动创建：
```
我的知识库/
├── 每日计划/
├── 每周计划/
└── 任务/
```

**示例 2：使用平级结构（不嵌套）**

配置：
```json
{
  "basic": {
    "folders": {
      "root": "",
      "structure": {
        "dailyPlan": { "name": "日计划", "pattern": "{date}" },
        "weeklyPlan": { "name": "周计划", "pattern": "W{week}" },
        "topic": { "name": "事项" }
      }
    }
  }
}
```

结果：
```
仓库根目录/
├── 日计划/
├── 周计划/
└── 事项/
```

## 七、常见问题解答

### Q1: 为什么打开工作台后出现了「SAP 学习」「项目任务」这些分类？

**来源**：周计划模板中的 `## 本周目标` 分类

**配置项**：`advanced.templates.weeklyCategories`，默认值为 `["SAP 学习", "项目任务", "写作/事项"]`

**修改方式**：在 `config.json` 中修改：
```json
{
  "advanced": {
    "templates": {
      "weeklyCategories": ["我的分类1", "我的分类2"]
    }
  }
}
```

### Q2: 为什么创建日计划会自动创建周计划？

**触发条件**：创建日计划时，如果本周的周计划不存在，会弹窗询问是否创建。

**如何避免**：先手动创建周计划，或者在弹窗时选择「取消」。

### Q3: 为什么周计划里会自动出现日计划的链接？

**触发条件**：创建日计划时，会自动在周计划的「每日进展」部分添加链接。

**如何避免**：这是设计行为，无法关闭。如果不想关联，可以手动删除周计划中的链接。

### Q4: 为什么会出现「逾期事项.md」「领域统计-AI.md」这些文件？

**触发条件**：点击状态栏卡片（逾期/已完成/进行中）或领域卡片时会自动生成。

**如何避免**：不点击这些卡片就不会生成。如果已经生成，可以手动删除。

### Q5: 如何修改「任务 1」「内容 1」这些默认内容？

**来源**：模板文件 `src/utils/templates.ts`

**修改方式**：需要修改源代码中的模板函数，然后重新构建插件。

**未来改进**：计划支持用户自定义模板（通过配置文件）。
